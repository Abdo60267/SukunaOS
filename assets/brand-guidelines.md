# SukunaOS Brand Guidelines

## Direção

SukunaOS deve parecer um sistema operacional sério tomado por uma identidade de domínio: escuro, preciso, agressivo na composição e ainda legível para uso diário. O visual não deve ser apenas "Linux gamer vermelho"; ele precisa carregar marcas, cortes, selos e a sensação de um altar digital.

## Cores

- Fundo abissal: `#050203`
- Preto ritual: `#0b0304`
- Vermelho sangue: `#b30d18`
- Vermelho profundo: `#69020a`
- Ouro selo: `#e6b84d`
- Osso claro para texto: `#f4e6cf`
- Cinza carneiro para apoio: `#b99c84`

## Motivos Visuais

- Marcas faciais abstratas: linhas vermelhas inclinadas, sempre simétricas ou quase simétricas.
- Cortes `Cleave/Dismantle`: faixas diagonais finas, com núcleo vermelho e filete dourado.
- Selos de domínio: círculos concêntricos, pequenos pontos dourados e linhas radiais.
- Altar/Shrine: silhuetas centrais com base pesada, bordas douradas e interior escuro.
- Textura: grades sutis e linhas de auditoria para lembrar um sistema real.

## Tipografia

- Primária: sans-serif moderna e profissional.
- Secundária: monoespaçada para terminal, logs, build e segurança.
- Títulos podem usar peso forte, mas sem exagero decorativo.

## Uso do Logo

- Use o brasão principal em telas de boot, splash, installer e capa de documentos.
- Em ícones pequenos, use apenas o quadrado com marcas vermelhas e selo dourado.
- Não substitua o brasão por triângulos genéricos; a identidade precisa mostrar marcas/cortes.

## Wallpapers

- `wallpaper-1.svg`: conceito Malevolent Shrine, ideal para login, boot e marketing.
- `wallpaper-2.svg`: conceito Cleave/Dismantle, ideal para desktop padrão e mockups.
- Sempre preserve contraste suficiente para painéis e texto sobrepostos.

## Interface

- Componentes principais usam fundo quase preto, borda vinho e estados ativos em ouro.
- Modos e ações podem usar nomes temáticos quando a clareza não for prejudicada:
  - `Domain Control` para central de controle.
  - `Cleave Guard` para proteção ativa.
  - `Dismantle Scan` para análise.
  - `Shrine Snapshots` para rollback.

## Sons

- Graves curtos, metálicos e discretos.
- Sons de erro podem usar uma batida seca; sucesso deve ser um selo leve, não fanfarra.
- Deve haver opção para desativar todos os sons.
